# E017: D4 Categorical Formalization

## Purpose

RUN-0039 left a primary next target: define a category of issuance-capable systems
(`IssSys`) and a category of closed systems (`CloSys`), then prove or refute the proposed
categorical witness:

> no fully faithful functor from `IssSys` to `CloSys` exists.

Success would move D4 toward a formal categorical separation. Failure would weaken the claim
that D4 is more than closed-system bookkeeping.

## D4 Restated

D4 says a process exhibits issuance iff:

1. it produces states that are not instances of any type admissible before the production
   event; and
2. the production is not determined by any prior state together with any fixed transition rule
   over the existing type-schema.

The key discriminating feature in RUN-0039 was type-novelty. Non-determination prevents the
new type from being only the output of a fixed schema-expansion rule.

## Candidate Categories

### `CloSys`

An object of `CloSys` is a closed typed transition system:

```text
C = (Sigma, X -> Sigma, T)
```

where:

- `Sigma` is a fixed type schema;
- `X -> Sigma` assigns each state to a type in that fixed schema;
- `T` is a fixed deterministic, stochastic, relational, or otherwise rule-governed transition
  structure over `X`.

A morphism `f: C -> C'` is a pair of maps on schemas and states that preserves typing and
transition structure.

### `IssSys`

An object of `IssSys` is a prefix-indexed type-growing system:

```text
I = (Sigma_n, X_n -> Sigma_n, T_n, i_n, W_n)_{n in N}
```

where:

- `i_n: Sigma_n -> Sigma_{n+1}` and `X_n -> X_{n+1}` preserve prior admissible types and
  states;
- at selected stages, `Sigma_{n+1}` contains a new type `alpha_n` not in `Sigma_n`;
- `W_n` is the D4 witness that `alpha_n` is not generated by any allowed fixed rule over
  `Sigma_n`.

A morphism `h: I -> J` is a compatible family of prefix maps preserving prior schemas,
issuance stages, and D4 witnesses.

## Three Functor Attempts

### Attempt 1: Prefix Forgetful Functor

Send an issuing system to one closed prefix:

```text
F_k(I) = (Sigma_k, X_k -> Sigma_k, T_k)
```

This is not faithful to D4. It forgets every later issuance event, and two systems that agree
through prefix `k` but diverge later become identical. It cannot serve as the intended
categorical witness.

### Attempt 2: Completed-History Encoding

Send an issuing system to a closed system whose fixed schema contains the entire completed
history:

```text
U(I) = (Sigma_infty + stage-tags + witness-tags, Hist(I), shift)
```

This absorbs D4 into a fixed higher-level schema. The state of the closed system is no longer
"current world state under the current schema"; it is the whole completed issuance history,
tagged enough to recover stages and witnesses.

If `CloSys` is allowed to include such completed-history encodings, the unqualified no-go
claim fails. The issuing object can be represented as a closed object with a larger fixed
schema. The apparent issuance becomes internal data of that closed object.

This is not a satisfying physical explanation, but it is enough to block the proposed
unqualified theorem. The categorical problem is exactly PP-1 in another form: a meta-level
closed schema can encode object-level type novelty.

### Attempt 3: Online / Prefix-Faithful Embedding

Restrict the target to closed systems that are online and prefix-faithful:

1. no hidden future schema is allowed at prefix `n`;
2. type admissibility at prefix `n` must be reflected by the target schema at prefix `n`;
3. a target transition rule at prefix `n` may only use `Sigma_n`;
4. morphisms must preserve the distinction between "not yet admissible" and "already
   admissible but unobserved."

Under these restrictions, a D4 issuance event cannot be represented as a closed transition
inside the prefix schema.

Proof sketch:

Let `I` issue a new type `alpha` at stage `n+1`, with `alpha notin Sigma_n`. Suppose an
online, prefix-faithful closed representation `F(I)` maps the issuance event to a transition
inside a closed prefix object over schema `Gamma_n`. Prefix faithfulness reflects admissible
types, so `alpha` cannot already be represented in `Gamma_n`. But the closed transition target
must be typed in `Gamma_n`. Contradiction.

This is a valid conditional obstruction, but it is not the original broad theorem. It depends
on adding the no-hidden-schema and prefix-faithfulness constraints.

## Verdict

The unqualified categorical witness is killed:

```text
No fully faithful functor IssSys -> CloSys exists.
```

That claim is false or at least unearned for broad enough definitions of `CloSys`, because
completed-history or hidden-global-schema encodings can absorb object-level issuance.

What survives is narrower:

```text
No online, prefix-faithful, admissibility-reflecting closed-system representation of a D4
issuance event exists without either (a) adding a hidden future schema, (b) moving issuance to
a meta-level rule, or (c) losing the distinction between not-yet-admissible and merely
unobserved.
```

This is useful, but it weakens RUN-0039's hoped-for categorical separation. D4 is not
categorically irreducible unless the target category forbids hidden ambient schemas.

## Effect On TI-C019

TI-C019 is weakened but not killed.

D4 still discriminates issuance from the named classical competitors in RUN-0039. The
categorical pass shows that D4 does not yet defeat a stronger absorber: closed-system
encoding at the meta-schema level. This is the categorical version of PP-1.

The next hard work is not another broad category proof. It is to state the exact no-hidden-
schema / no-anticipation constraint and test whether it is principled rather than smuggled in
to protect D4.

## Next Target

Run the PP-1 self-reference test:

1. define `MetaCloSys`, a closed system whose fixed schema generates object-level type
   schemas;
2. define `MetaIssSys`, where the meta-schema itself can issue type-novel generators;
3. ask whether D4 at object level always becomes closed update at some sufficiently broad
   meta-level, or whether an infinite regress forces issuance at the base of the tower;
4. record the result as either absorption by meta-stochastic update or a principled
   no-hidden-schema constraint.

Secondary: use the Cech/sheaf fixture only after the PP-1 constraint is explicit. Otherwise a
new cocycle class can always be reclassified as already present in a larger fixed sheaf
schema.
