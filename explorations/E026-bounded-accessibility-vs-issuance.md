# E026: Bounded Accessibility vs Source-Side Issuance

## Purpose

Two versions of the same note were synthesized. Both ask the same question from slightly
different angles: after RUN-0039 through RUN-0041, has the center of gravity of the TI
program quietly shifted from "issuance" to "access"?

This exploration records the question without resolving it. No claim statuses are updated here.
The question is pressure, not evidence.

**What was filtered out:** The GU/14D shadow-projection framing (Note 2) is explicitly
noted as a grounding thought experiment, not a formal claim. Using a 14D observerse as a
physical hypothesis would import unearned physics. The abstract version of the same idea
(richer source structure → bounded projection → observer-level schema) does not require GU
and is the formalism-safe formulation. The GU framing is recorded below for conceptual
orientation only — it is NOT integrated as a formal object.

## The Center-of-Gravity Shift

RUN-0039 through RUN-0041 produced a sequence of narrowings:

```text
RUN-0039: D4 discriminates issuance from stochastic update, information flow, causal-set
          growth, and thermodynamic entropy. Discriminator survives.

RUN-0040: A broad enough closed-system category can encode completed D4 traces in a fixed
          meta-schema. Object-level type novelty is absorbed at meta-level.

RUN-0041: PP-1 self-reference argument fails. MetaCloSys can absorb finite D4 traces
          without generating an infinite regress.

that do not have access to hidden future schema.
```

The surviving distinction is:

```text
NOT: issuance-capable systems vs. non-issuance-capable systems
BUT: systems with access to future schema vs. systems without access to future schema
```

These are not the same thing. The notes point out that this shift may be decisive.

## The Core Hypothesis Pair

### Hypothesis A: Source-Side Issuance

New possibility genuinely enters reality. Issuance is not merely disclosure of
inaccessible structure — it is not reducible to a bounded observer receiving schema that
was already "there."

```text
Issuance → new source-side type → observer-accessible novelty
```

Under Hypothesis A, D4 detects something real at the source level. The observer's inability
to access future schema reflects actual non-existence of that schema before the issuance event.

### Hypothesis B: Bounded Accessibility / Shadow Projection

Reality may contain richer structure than observers can access. What appears as issuance is
the progressive disclosure of structure that was not accessible from the prior observer domain
— not absent from reality, only invisible.

```text
Richer source structure
    → bounded projection (limited observer access)
    → observer-accessible schema
    → apparent novelty
```

Under Hypothesis B, D4 detects observer-relative or projection-relative novelty. OnlineSchemaSys
correctly models the observer's bounded-access process. But the source may be a static MetaCloSys
from which only partial projections are accessible at each step.

**Key point:** Under Hypothesis B, OnlineSchemaSys is the right model of the observer's process,
but not a model of the source. The source could be a completed MetaCloSys projecting through a
time-varying aperture. This would preserve all OnlineSchemaSys structure while denying that
source-side issuance is real.

## The Relation to OnlineSchemaSys

This is the critical formal question the notes raise:

**Does OnlineSchemaSys already model bounded accessibility, or does it model genuine source-side issuance?**

Under Hypothesis B, OnlineSchemaSys is already the right formal model — it models a bounded-access
observer process. But TI-C019 claims the physical universe IS an OnlineSchemaSys instance (source-side).
If Hypothesis B is correct, TI-C020 should say instead: "observers inhabit an OnlineSchemaSys
process, but the underlying reality may be a MetaCloSys."

This is not a kill condition for OnlineSchemaSys or D4. It is a reinterpretation:

| Current claim | Hypothesis A | Hypothesis B |
|---|---|---|
| Physical universe is OnlineSchemaSys | Source is OnlineSchemaSys | Source is MetaCloSys; observers access it as OnlineSchemaSys |
| D4 detects source-side issuance | D4 detects source-side novelty | D4 detects observer-access novelty |
| NAA is principled by M2/M3 | NAA reflects source structure | NAA reflects observer access limits |

## Novelty Decomposition (Five Types)

The notes introduce a useful decomposition. The word "novelty" may cover at least five
distinct things, only some of which require source-side issuance:

| Type | Description | Requires source-side issuance? |
|---|---|---|
| Observer discovery | Observer accesses schema already present in shared state | No |
| Community synchronization | Schema becomes accessible to a community via record propagation | No |
| Schema disclosure | Observer's projection expands to expose schema from richer source | No (under Hyp. B) |
| Information acquisition | Observer updates beliefs about existing schema | No |
| Genuine type creation | New type enters shared source-side process | Yes (required for Hyp. A) |

D4, as currently formulated, detects events where the new type is not derivable from any
rule over the prior schema. But this detection operates at the observer-accessible schema
level, not at the source level. D4 is consistent with both Hypothesis A and Hypothesis B.

**The precision gap:** D4 does not distinguish genuine type creation from schema disclosure.
The notes identify this as the primary formalism gap.

## Nested Observer Domains

The notes propose that "the system" should not be treated as flat. Reality may contain
nested accessibility domains:

```text
Source structure (richer / more complete)
    ↓
Regional projection / accessible world slice
    ↓
Civilization-level records
    ↓
Community-level synchronized state
    ↓
Individual observer accessible schema
```

At each level, "new" means something different. A new type at one level may be:
- Known at higher levels (schema disclosure downward)
- Unknown at higher levels (genuine upward issuance)
- Unknowable at any level (type-novel even to the source)

The repo has been operating primarily at the level of "the system" without specifying which
level is the target of TI-C019. RUN-0040/0041 attacked the strongest version (universe-level
source-side issuance) and found it hard to earn. The notes suggest the program should specify
which level is the primary formal target.

## The Shadow-Projection Thought Experiment (Framing Only)

**This is a framing device, not a formal claim. The GU/14D physics is NOT integrated.**

The abstract version of the thought experiment:

> Suppose reality has a richer structure (call it the source layer) than any embedded observer
> can access. Observers access a bounded projection of this structure. When a new type appears
> in the observer-accessible projection, there are two interpretations:
>
> Interpretation 1 (Issuance): The type was not present in the source layer. It genuinely
> entered the shared process. This requires source-side novelty.
>
> Interpretation 2 (Projection Disclosure): The type was already present or implied in the
> richer source structure, but the observer's projection did not previously expose it. This
> does not require source-side novelty.
>
> The question for TI: what would distinguish these two interpretations observationally?

This is exactly the D4 precision gap stated differently. D4 as currently formulated does not
answer this question — it detects schema novelty in the projection, not in the source.

If the source layer is a static MetaCloSys and projection is expanding over time, then:
- NAA holds (projection cannot anticipate unexposed source structure)
- OnlineSchemaSys correctly models observer process
- D4 events are real from the observer's perspective
- But source-side issuance is an illusion — it is disclosure, not creation

This would not kill D4 or OnlineSchemaSys. It would reinterpret them as observer-process
models rather than source-level reality models.

## What the Formal Test Requires

The notes propose a bounded-accessibility model with at least two layers. The formal requirements
(formalism-safe version):

**Source layer:**
- A formal object S_∞: a completed (potentially MetaCloSys) or partially-complete type space
- S_∞ may be unknown and unknowable to any single observer

**Projection layer:**
- Observer-accessible schema S_n ⊆ π(S_∞) where π is a projection/access function
- Projection may expand over time as n increases
- Observers may not know the extent of S_∞ from S_n alone

**Tests:**

1. Can D4 events occur in the projection without S_∞ itself expanding?
   - If yes: D4 detects projection-relative novelty, not source-relative novelty
   - If no: D4 requires source expansion — genuine issuance required

2. Does NAA follow from projection limits alone (without source-side issuance)?
   - If yes: NAA reflects observer epistemic limits, not source structure
   - If no: NAA requires source-side sequentiality

3. Does the morphism non-embedding of OnlineSchemaSys into MetaCloSys hold when the MetaCloSys
   is the source and OnlineSchemaSys is the projection process?
   - This is the sharpest version: is the non-embedding a fact about the source, or about
     the observer's access to the source?

4. What would count as a positive witness for source-side issuance (as distinct from
   projection disclosure)?
   - This is the key operational question the repo has not yet answered.
