# E080: Dual-Record Opportunity Hypothesis Steelman Vote

## Intake

Joe proposed a weak hypothesis:

> Maybe the important thing about two different manifolds, layers, or systems of records is not
> the specific 4D/14D, Standard Model/quantum, or manifold split. Maybe the important thing is
> that one record system creates explorative opportunities that let a Quantum-Darwinism-like or
> evolutionary process make larger jumps when it finds local minima. The same pattern may appear
> in cultural evolution, biology, and other information systems.

This run steelmans that idea without making the physical manifold examples load-bearing.

## Baseline Discipline

Do not claim:

```text
two manifolds prove issuance
quantum layer proves open-ended source growth
cultural evolution proves physics
local-minimum escape implies source-side issuance
```

Allowed candidate:

```text
Some systems may maintain two record regimes:
  1. a stabilized public/action record used for exploitation, and
  2. an exploratory opportunity record that preserves unrealized options, variants, or move
     operators.

The second regime may improve escape from local minima.
```

The main absorber is obvious:

```text
fixed latent search space + stochastic exploration + access schedule
```

Any surviving version must beat that absorber.

## Five Steelman Versions

### V1: Opportunity-Reservoir Dual Record

The system has two records:

```text
R_stable: records what has become actionably settled
R_opp: records not-yet-finalized opportunities, variants, or unrealized affordances
```

`R_stable` supports exploitation and shared finality. `R_opp` preserves option diversity and
candidate moves that are not yet public/actionable. Local-minimum escape improves when
`R_opp` can re-enter search as new move proposals.

Best formal target:

```text
R_opp changes the proposal kernel K_n over moves while R_stable preserves selected outcomes.
```

Kill risk:

```text
R_opp is just a fixed latent option pool or random-restart memory.
```

### V2: Compression / Innovation Dual Ledger

One record layer compresses the world into stable reusable summaries. A second layer keeps a
higher-dimensional or less-compressed generative ledger. Exploration comes from the mismatch:
the stable layer is efficient but local-minimum-prone; the generative layer can re-expand
possibility and suggest jumps not visible in the compressed representation.

Best formal target:

```text
compression map C: R_gen -> R_stable
loss kernel L = information lost by C
innovation occurs when R_gen supplies a move using data lost under C
```

Kill risk:

```text
standard lossy compression plus latent-variable search absorbs the whole story.
```

### V3: Error-Corrected Exploitation / Noisy Exploration Split

The stable record layer acts like error correction: it protects winning structures from noise.
The exploratory layer remains noisier, more plastic, or less finalized, allowing variation.
Darwinian selection then filters exploratory variants into stable records.

Best formal target:

```text
stable layer = low-temperature / high-fidelity memory
exploratory layer = high-temperature / variation memory
selection channel = transfer from exploratory to stable layer
```

Kill risk:

```text
this is simulated annealing, evolutionary search, QEC, or standard exploration-exploitation
with new names.
```

### V4: Adjacent-Possible Record Manifold

The second record system does not merely store more states. It stores adjacency relations:
what moves, recombinations, or transformations are admissible next. Local-minimum escape
depends on changing the adjacency graph, not just moving through an existing graph.

Best formal target:

```text
G_stable,n = graph of finalized states
G_opp,n = graph of admissible next moves / recombination edges
issuance-like event = edge or move-operator appears that was not present in G_opp,n
```

Kill risk:

```text
the new edge was already in a fixed meta-graph and only became accessible.
```

### V5: Multi-Scale Evolutionary Record Stack

Biology, culture, and cognition may use stacked records: genotype/phenotype, private/public
beliefs, draft/published institutions, speculative/stabilized theories. The common pattern is
not quantum mechanics but information architecture: an exploratory record stack generates
candidate variants, and a stabilized record stack preserves selected variants.

Best formal target:

```text
two-level record stack with transfer rule T: exploratory variants -> stabilized inheritance
```

Kill risk:

```text
this is ordinary evolutionary search, cultural selection, or multi-level selection without
source-side novelty.
```

## Winning Steelman

The best version is:

```text
Dual-record systems matter when one record regime stabilizes selected outcomes and the other
maintains or changes the adjacency graph of future admissible moves.
```

Plain English:

```text
One record says "what we can currently rely on."
The other says "what new moves are becoming possible."
The important part is not two manifolds. It is the split between stable exploitation records
and opportunity-edge records.
```

This version is strongest because it creates a discriminator:

```text
Does the second record system merely reveal moves from a fixed latent graph,
or does it change the admissible move graph itself?
```

That aligns with the repo's existing source/projection split.

## Strongest Absorbers

1. **Fixed latent graph absorber**

   All future moves exist in `G_infty`; the opportunity record only reveals them.

2. **Simulated annealing absorber**

   Larger jumps come from temperature/noise/random restarts, not a second record system.

3. **Evolutionary search absorber**

   Variation plus selection already explains local-minimum escape.

4. **Bayesian nonparametric absorber**

   New apparent options come from a fixed hyperprior/generative grammar.

5. **Quantum Darwinism absorber**

   Quantum/classical record split explains objectivity, not source-side opportunity growth.

6. **Cultural evolution absorber**

   Private drafts and public records are ordinary multi-level selection.

## Fixture Candidate

Next candidate fixture:

```text
dual_record_adjacent_possible_fixture
```

Model:

```text
S_n = stabilized public/action record
O_n = opportunity record over admissible move edges
G_n = current move graph
K_n = proposal kernel induced by O_n
T_n = transfer/finality rule from opportunity to stable record
```

Compare three systems:

```text
A. Single-record search:
   stable record plus ordinary mutation/noise

B. Dual-record fixed-latent search:
   stable record plus opportunity access to fixed G_infty

C. Dual-record growing-adjacency search:
   stable record plus O_n events that add move operators or edges not present in G_n
```

Positive condition:

```text
C escapes local minima in a way not reproducible by A or B under equal search budget,
and the new move edge cannot be represented as bounded access to fixed G_infty.
```

Failure condition:

```text
Every advantage is reproduced by fixed latent graph, annealing, random restarts,
evolutionary search, or Bayesian nonparametric generation.
```
