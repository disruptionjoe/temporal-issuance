# E125: Capability-Typed Ext_S Surplus Test

## Purpose

Execute the next gate from E124:

```text
capability_typed_Ext_S_surplus_test
```

The test asks whether a region-indexed `Ext_S` rule changes what an observer
or bounded region can do in a way that is not already expressible as fixed
block completion, CSG-style covariant growth, fixed-source aperture
disclosure, or TaF capability-typed readout.

This is an adversarial surplus test, not a claim-promotion pass.

## Region And Task Menu

Let `R` be a bounded observer/support region with boundary `dR`. Let
`Records_R,n` be the records available to the holder of the region at prefix
`n`, and let `Access_R,n` be the currently exposed part of a possible richer
source.

Define the region task menu:

```text
TaskMenu(R, n) =
  {
    observe(q): answer a record-supported query q,
    postdict(e): infer or certify a past boundary event e,
    choose(a): select an available intervention or continuation a,
    certify(r): finalize or quorum-certify record r,
    construct(w): form an admissible witness, predicate, constructor, or task
                  type not available in the prior source context
  }
```

Only the last task kind can carry `Issue[S]` pressure. The first four are
capability changes, but they can be fully real and still be projection,
readout, or finality changes rather than source issuance.

## Candidate Surplus Condition

A capability-typed `Ext_S` surplus would require all five conditions:

1. **Region support:** the task-menu change is stated for `R` without a global
   present, preferred foliation, or hidden global slice.
2. **Source-side formation:** `construct(w)` becomes well-typed because a new
   source admissibility predicate, constructor, or witness relation is formed.
3. **Record preservation:** the before/after trace records what changed in a
   way another reviewer can audit.
4. **Absorber defeat:** the change cannot be represented as fixed block
   completion, CSG-style fixed-law growth, fixed-source aperture disclosure, or
   TaF capability readout.
5. **No cross-repo identity claim:** the comparison to TaF is a pressure test,
   not evidence for Temporal Issuance and not an identity claim.

If a proposed `Ext_S` change only says "the holder of `R` can now do X," then
TaF-style capability readout already expresses the honest ledger. Temporal
Issuance needs the extra source-formation reason why X became possible.

## Absorber Comparisons

| Candidate capability change | Fixed block completion | CSG-style covariant growth | Fixed-source aperture | TaF capability readout | TI surplus verdict |
| --- | --- | --- | --- | --- | --- |
| A new record becomes observable in `R` | Absorbs as region-limited access to a completed structure | Absorbs if the event is generated by a fixed covariant law | Absorbs as `Access_R,n` widening | Expresses the capability change directly | No `Issue[S]` |
| A holder can postdict a boundary event | Absorbs as using already-present block relations | Absorbs as ordinary ancestry/order inference | Absorbs as delayed exposure of source data | Expresses postdictive capability | No `Issue[S]` |

## Result

The capability-typed readout comparison is stronger than the older
statistics-only comparison. It prevents a false positive: a real task-menu
change does not by itself establish `Issue[S]`.

The current region-indexed `Ext_S` candidate does not supply a concrete new
source admissibility predicate, constructor, or witness relation beyond the
formal `OnlineIssuance^LC` class. Therefore the capability-reading surplus is
absorbed as readout/access/finality vocabulary for this physical or
cross-repo-facing gate.

This does not disturb `OnlineIssuance^LC`. It says only that region capability
change is not enough. Temporal Issuance must show source formation, not merely
capability difference.

## What Collapsed

- Statistics-only `Ext_S` comparison as the right adversarial target.
- Any inference from "region `R` can do more" to `Issue[S]`.
- Any use of TaF-style capability change as support for Temporal Issuance.
- Any global-present or objective-passage residue from the block-universe
  branch.

## What Survives

The strict source-formation burden survives:

```text
State a region-supported source admissibility predicate, constructor, or
witness relation that becomes formed across `Ext_S`, preserves an audit trace,
and cannot be represented as fixed block completion, CSG-style fixed-law
growth, fixed-source aperture disclosure, or capability readout.
```

That burden is exactly where `OnlineIssuance^LC` remains useful. It is a
formal/class-relative source gate, not a physical or cross-repo identity
result.

## What Would Reverse This Verdict

Reopen the capability-surplus branch only if a future fixture supplies:

- a specific region `R`;
- a task menu with at least one `construct(w)`-type task;
- the exact new admissibility predicate, constructor, or witness relation;
- an audit-preserved before/after trace;
- proof that the task was not in fixed block completion, not generated by a
  fixed CSG law, not precontained in `Mu_infty`, and not merely TaF-expressible
  capability readout;
- no identity claim across TI/TaF/GU.
