# E006: Ontology And Survivor Competition Results

## Purpose

Record the fixture-based ontology and survivor competition required after RUN-0022.

This result uses `HYPOTHESIS-STEELMAN.md` as the strongest target and `explorations/E005-ontology-competition-fixture-suite.md` as the shared test surface.

## Steelman Target

The strongest live Temporal Issuance target is:

```text
source-side constraints become admissibly extended,
and observer histories are lossy projections of that extension.
```

The target is not a global moving present, observer-created reality, or a bare source order. The competition asks whether any source-side residue beats `NULL-SURVIVOR`.

## Competition Rule

A survivor does not win by being the last vocabulary left standing. It must provide at least one of:

- a witnessable hidden-source distinction
- an invariant that survives representation or gauge quotienting
- a transformation consequence not reduced to entropy, information, cost, action, probability, volume, proof, computation, or known resource convertibility
- a formal discriminator that existing absorbers do not already explain as well

## Source-Order Taxonomy Result

The source-order taxonomy in E003 does not identify a surviving independent order primitive.

| Candidate meaning | RUN-0024 result |
| --- | --- |
| causal order | absorbed by relativity or causal-set-style order |
| dependency order | absorbed by ordinary dependency graphs, build/proof dependencies, or DAGs |
| constraint order | demoted to a possible shadow of `Ext_S` |
| realization order | circular unless `Ext_S` defines realization without time or records |
| proof order | absorbed by proof or type theory |
| computational order | absorbed by algorithms, rewriting, or execution traces |
| construction order | absorbed by constructor or resource theory unless new transformability boundary appears |
| observer-independent order | hidden-variable risk unless witnessable and quotient-stable |
| observer-relative order | absorbed by time-as-finality, access, or record theory |
| resource preorder | absorbed unless free operations and monotones differ from known resource theories |
| projection kernel order | better treated as projection loss than order |
| null order | currently strongest against bare `<=_S` |
