# E127: OnlineIssuance LC c.e. Tier Comparator

## Purpose

Execute the strict c.e. / partial-computability comparator left open by E121
and selected by E126.

E121's `Comparator.lean` proved the diagonal escape / absorption / no-fixed-
point dialectic for total Nat-indexed families. It explicitly did not model
true c.e. generation, enumeration order, non-output, partiality, a universal
machine, or s-m-n. E126 ranked that gap as the highest-value next pressure
target.

This run adds:

```text
formal/lean/OnlineIssuance/CEComparator.lean
```

It proves a bounded c.e.-presentation result in core Lean: finite-prefix escape
survives, but whole-c.e. positive escape is impossible because any proposed
candidate is generated by its singleton c.e. enumerator.

## Model Boundary

Chosen model:

```lean
abbrev CEEnumerator (alpha : Type u) := Nat -> Option alpha
def CEGenerates (E : CEEnumerator alpha) (a : alpha) : Prop :=
  Exists fun n => E n = some a
```

What it captures:

- stagewise enumeration;
- explicit non-output at a stage via `none`;
- finite-prefix observations of a stream;
- the quantifier-order ceiling that defeats "escapes every c.e. enumerator"
  claims.

What it does not capture:

- recursive-function theory;
- a universal machine;
- s-m-n;
- a proof that the enumerator itself is computable;
- a mathlib `Partrec`-grade theorem over partial recursive codes.

This is still enough to settle the next routing question: a positive c.e.-tier
escape theorem cannot be the next honest strengthening unless the model also
adds an internal computability/code discipline that forbids after-the-fact
singleton naming.

## Interpretation

The strict c.e. tier does not strengthen `OnlineIssuance^LC` by producing a
candidate that escapes every c.e. generator. That target has the wrong
quantifier order:

```text
for every candidate a, singletonCE(a) generates a
```

So an absolute c.e.-escape claim is absorbed immediately as after-the-fact
singleton enumeration. The positive residue remains prefix-relative:

```text
given only the finite observed prefix of a c.e. stream, a fresh name can be
constructed outside that prefix
```

Then the whole-family level absorbs that fresh output once the singleton
enumerator is allowed.

This sharpens, rather than promotes, the formal residue:

- E121's total-family diagonal theorem stays valid inside its stated model.
- E127 says the strict c.e. whole-family target is not the right next positive
  theorem unless a future module models internal computability/code syntax
  tightly enough to forbid singleton after-naming.
- No physical source issuance follows.
- `TI-C020` remains parked.
