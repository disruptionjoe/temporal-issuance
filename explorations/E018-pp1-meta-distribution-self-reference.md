# E018: PP-1 Meta-Distribution Self-Reference Test

## Purpose

RUN-0040 left PP-1 as the load-bearing pressure point for D4. Object-level type novelty can be
absorbed by a broader closed meta-schema if that meta-schema already contains the possible
future types, type generators, or completed issuing history.

This pass tests whether that absorption always wins, or whether an infinite regress forces a
base-level issuance primitive.

## Starting Point

D4 says a process exhibits issuance iff:

1. it produces states that are not instances of any type admissible before the production event;
2. that production is not determined by prior state plus any fixed transition rule over the
   existing type-schema.

RUN-0040 showed that the unqualified categorical no-go fails. A sufficiently broad closed
system can encode completed issuing history in a fixed meta-schema. The remaining question is
whether that move only relocates issuance to the meta-level or genuinely absorbs it.

## Candidate Systems

### `MetaCloSys`

A `MetaCloSys` object is a fixed meta-schema generator:

```text
M = (Mu, ObjSchema, Gen, K)
```

where:

- `Mu` is a fixed meta-schema of schema descriptions, type constructors, generator tags, and
  witness tags;
- `ObjSchema` is the class of object-level schemas representable inside `Mu`;
- `Gen` is a fixed set of generator rules or probability kernels over `ObjSchema`;
- `K` is a fixed transition, sampling, or update rule over meta-states.

At the object level, `M` may generate schemas that look type-novel relative to a prior
object schema `Sigma_n`. At the meta-level, they are not type-novel if they are all instances
of descriptions already admitted by `Mu`.

### `MetaIssSys`

A `MetaIssSys` object is a prefix-indexed meta-schema system:

```text
I = (Mu_n, ObjSchema_n, Gen_n, K_n, W_n)_{n in N}
```

where:

- `Mu_n` is the current meta-schema;
- `ObjSchema_n` contains the object schemas currently representable by `Mu_n`;
- `Gen_n` contains the generator kinds currently admissible;
- at some stage `n+1`, a generator kind `g_new` or meta-type `m_new` becomes admissible even
  though it was not an instance of any type in `Mu_n`;
- `W_n` witnesses that `g_new` or `m_new` is not determined by prior meta-state plus a fixed
  rule over `Mu_n`.

This is D4 applied to the meta-level itself.

## Strongest Closed Meta-Schema Absorption

The strongest absorber is not ordinary stochastic update over object states. It is a closed
system over descriptions of possible object schemas:

```text
fixed Mu = all schemas generated by grammar G
K = sample or compute next object schema description in Mu
```

Then a new object type `alpha notin Sigma_n` is not issuance at the meta-level. It is an
instance of a schema-description type already present in `Mu`.

This defeats any claim that object-level type novelty alone is enough. If the comparison class
allows omniscient or completed-description meta-schemas, finite object-level D4 traces are
always encodable as closed meta-update.

## Self-Reference Pressure

The tempting pro-D4 move is:

```text
If Mu generates object schemas, ask what generated Mu.
If Mu_1 generated Mu_0, ask what generated Mu_1.
Therefore issuance is forced at the base.
```

This does not work as a theorem. A closed theory can stop the regress by treating `Mu` as a
fixed universal grammar, a completed mathematical domain, or a primitive state space. That may
be metaphysically unsatisfying, but it is not formally inconsistent.

The regress becomes live only under an operational constraint:

```text
At prefix n, the system may use only schema descriptions, generator kinds, and admissibility
tests constructible from the current prefix state.
```

Under that constraint, hidden future generators are disallowed by definition. But then the
constraint, not D4 alone, does the work. The question becomes whether the constraint is
principled or merely protective.

## Dichotomy

PP-1 produces a clean dichotomy.

### Option A: Completed-Domain Reading

All possible schemas, generators, and histories are admitted inside a fixed `Mu`.

Result:

```yaml
object_level_D4: absorbed
meta_level_issuance_required: false
cost: type_novelty becomes relative to observer/object prefix, not primitive
```

This option kills any absolute D4 irreducibility claim.

### Option B: Online Constructive Reading

Only prefix-available schema descriptions and generator kinds count as admissible.

Result:

```yaml
object_level_D4: survives_class_relatively
meta_level_issuance_required: possible_but_not_forced
load_bearing_constraint: no_hidden_future_schema / no_anticipation
cost: the constraint needs independent motivation
```

This option keeps D4 useful, but only as a class-relative operational discriminator.

## Verdict

PP-1 does not force a base-level issuance primitive. The infinite-regress argument is killed
as a proof strategy because a closed completed-domain or universal-grammar meta-schema can
stop the regress without contradiction.

What survives is narrower:

```text
D4 is a non-bookkeeping discriminator only relative to online, prefix-presented systems that
forbid hidden future schema, completed-history encodings, and meta-level rule import.
```

This is not empty. It is the same kind of class-relative restriction used in many no-go and
expressivity results: the theorem is about systems with a stated information boundary. But it
is weaker than the founding hope. D4 no longer looks like an absolute metaphysical primitive.
It looks like a candidate operational primitive for non-anticipatory schema growth.

## Effect On TI-C019

`TI-C019` remains weakened. It is not killed because the online constructive reading still
marks a real distinction between:

- state update inside a fixed present schema;
- object-schema generation from a fixed meta-schema;
- schema growth that is not available from the current prefix presentation.

But `TI-C019` is not promoted or strengthened. PP-1 shows that D4 cannot carry the North Star
claim unless the repo earns a principled no-anticipation constraint.

## What Was Killed

The path killed in this run is:

```text
PP-1 infinite regress as proof that D4 forces a base-level issuance primitive.
```

The local-minimum risk is real: killing the proof strategy must not kill the operational D4
distinction. The surviving route is class-relative and should be tested, not protected.

## Next Target

The next run should formalize the no-hidden-schema / no-anticipation constraint as an explicit
class-relative axiom:

```text
OnlineSchemaSys: prefix-presented schema systems whose transitions may use only currently
constructible schema descriptions, generator kinds, and admissibility tests.
```

Required test:

1. State the axiom without using the ordinary concept of future time as a primitive.
2. Show exactly which completed-history and universal-grammar encodings it excludes.
3. Decide whether the exclusion is principled for the Temporal Issuance research question.
4. Then run the Cech/sheaf fixture only inside that explicit class.

Failure condition: if the no-anticipation axiom cannot be stated without circularity or
protective stipulation, archive D4 as a useful object-level discriminator but not a primitive.
