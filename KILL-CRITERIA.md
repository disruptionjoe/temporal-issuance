# Kill Criteria

The repo succeeds if it quickly determines that the hypothesis should be killed.

## Kill Classes

### K1: Full Absorption

The hypothesis is fully absorbed by an existing framework, with no category mismatch and no residual explanatory work.

Required evidence:

- absorber named
- mapping supplied
- residual tested
- category-error risk assessed
- resurrection trigger recorded

### K2: Circularity

The hypothesis requires the ordinary concept of time to define temporal issuance, realization order, observer cadence, or frontier.

### K3: No Formal Work

The proposed object cannot produce a theorem, model, discriminator, pressure test, or useful failure condition.

### K4: No Distinguishable Claims

All claims reduce to known physics, known mathematics, or philosophy-of-time vocabulary without new explanatory leverage.

### K5: Metaphysical Residue

The surviving language cannot be made precise enough to support adversarial research.

## Path Kill Record

Every killed path must record:

```yaml
path:
reason_killed:
evidence:
local_minimum_risk:
possible_future_resurrection_trigger:
```

## Anti-Premature-Closure Rule

Before killing or absorbing a path, the steward must ask:

- Was the strongest version generated?
- Was the comparison category correct?
- Was a layer error possible?
- Would a heterodox persona object?
- What would resurrect this path?
