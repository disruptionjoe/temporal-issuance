# Category Layer Map

## Purpose

Prevent false absorption and false novelty by naming the layer of comparison.

## Layers

| Layer | Question | Example Absorbers | Failure Mode |
| --- | --- | --- | --- |
| Ontological | What exists or becomes fixed? | block universe, process philosophy | Treating a worldview as a formal model. |
| Physical | What does current physics already model? | relativity, thermodynamics, cosmological expansion | Treating known physical dynamics as new substrate. |
| Mathematical | What formal structure is already available? | causal set theory, gluing/restriction systems | Renaming an existing object. |
| Epistemic | What can observers know or inspect? | information theory, time-as-finality | Confusing access to records with reality becoming. |
| Operational | What procedure updates records or reconciles local views? | sampling, protocols, workflow analogies | Smuggling ordinary time through process steps. |
| Linguistic | What wording makes the idea feel stronger than it is? | process philosophy, metaphor, narrative | Mistaking compelling language for formal constraint. |

## W002 Use

For each `IssuanceSystem` component, W002 should name the layer of comparison before deciding whether the component is killed, absorbed, weakened, or preserved.

## Guardrail

Do not claim novelty from a layer mismatch. If an absorber and component operate on different layers, record the mismatch and test whether a bridging definition exists.
