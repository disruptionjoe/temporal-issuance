# GU Formalization

## Relationship

`gu-formalization` is not a Temporal Issuance absorber in the usual sense. It is prior-thinking context for how to handle no-go pressure, class-relative impossibility claims, specification obligations, and heterodox substrate hypotheses.

## Use

Consult GU formalization when:

- relativity, causal set theory, or information theory pressure is being treated as universal without checking assumptions
- the steward needs a specification discipline before defending a substrate claim
- a no-go result may be class-relative
- the repo risks protecting a vague substrate intuition rather than specifying it

## Boundary

Do not import GU claims into Temporal Issuance. Use the method lessons only:

- handle no-go claims by exact assumptions
- specify before defending
- preserve control cases
- use heterodox lenses as escape routes, not as proof
