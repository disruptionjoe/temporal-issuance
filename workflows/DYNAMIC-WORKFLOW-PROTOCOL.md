# Dynamic Workflow Protocol

The steward may create a dynamic workflow when an existing workflow is not the best fit for the current learning problem.

## Required Fields

```yaml
workflow_id:
status:
purpose:
trigger_conditions:
input_surfaces:
personas:
memory_pack:
expected_outputs:
merge_surfaces:
retirement_condition:
```

## Promotion

A dynamic workflow becomes durable only after repeated evidence shows that it improves verdict probability.

## Retirement

Retire workflows that no longer earn their keep. Retirement should preserve provenance and explain what replaced the workflow.
