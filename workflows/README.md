## Workflow Relationships

W007 (Steelman Pass) -> W009 (Heterodox Bridge Incubator) -> W008 (Bridge or Definitive Kill)

W007 builds the strongest version of the hypothesis before any adversarial testing.
W009 constructs the strongest possible mechanism for a bridge before Phase 6 failure analysis.
W008 audits escape routes from formal obstructions and applies the anti-local-minimum gate.
W010 scans current repo state and ranks the next most important work before W000 commits to a route.

Use W009 when a bridge is intuitively attractive but formally weak, before W008 is invoked.
Use W008 when formal obstructions have been recorded and escape routes need systematic audit.
Use W010 when multiple live next actions compete or when roadmap, memory, and next-trigger state may be stale or in tension.
