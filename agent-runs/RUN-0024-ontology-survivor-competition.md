## Steelman Before Attack

The strongest target is not source order by itself.

```text
source-side constraints become admissibly extended,
and observer histories are lossy projections of that extension.
```

The strongest component is `Ext_S`. `C` matters only if typed, and `<=_S` is killed as an independent primitive, surviving only as a possible derived invariant.

`NULL-SURVIVOR` beats all fully formulated competitors today. The only reason not to archive the remaining source-side residue immediately is that `Ext_S` now has a sharp, bounded next test: specify an admissible source-extension rule over typed constraints with a witness, invariant, transformation consequence, or source/readout discriminator not absorbed by existing frameworks.

## What Collapsed

Bare `<=_S` as an independent source-side primitive or primary survivor.

The fixture suite shows that the same order can carry different extension rules, that hidden order can vanish under quotienting, and that every bare order reading is absorbed by causal, dependency, proof, computational, construction, resource, observer-readout, or null order.

## What Survived

`Ext_S` survives only as a formalizing target. It is not promoted and does not yet beat archive.

`C` survives only as the typed carrier required by `Ext_S`; it does not survive as a standalone ontology.

Source order survives only as a possible derived invariant of a successful extension rule.

## What Was Absorbed

Most competitor lanes were absorbed into existing frameworks:

- source realization order into known order types unless extension does more work
- typed constraint inventory into event, record, proposition, proof, information, or boundary-data inventories
- projection kernel into ordinary information loss unless witnessable or invariant
- witness/certification into proof, cryptography, access, or provenance unless tied to source extension
- constructor/resource transformability into known resource, constructor, thermodynamic, or cost machinery
- obstruction class into sheaf or time-as-finality readout unless it constrains source extension
- proof/computation order into proof theory, type theory, formal methods, or rewriting
- emergence/lock-in into dynamical or complex-systems accounts unless it yields an independent source-extension invariant
